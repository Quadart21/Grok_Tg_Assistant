# Changelog

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/).

## [1.3.0] — 2026-07-21

### Добавлено

- Персональные расписания онлайна по аккаунтам в групповом чате
- Дружеские связи между выбранными сессиями с учётом этой динамики в генерации
- Возврат в диалог по расширенной истории после оффлайна
- Thread-aware reply/quote-логика с сохранением связей, кто кому ответил

### Изменено

- API и состояние группового чата поддерживают `account_schedules` и `friendships`
- LLM получает историю в формате reply-связей `A -> B`, чтобы не путать адресатов в параллельных ветках

### Исправлено

- Снижен риск коллапса диалога и недопонимания, когда несколько аккаунтов отвечают разным собеседникам одновременно

[1.3.0]: https://github.com/Quadart21/Grok_Tg_Assistant/releases/tag/v1.3.0

## [1.2.2] — 2026-07-21

### Изменено

- Вкладка **Групповой чат** разбита на 6 последовательных подразделов: состав, роли, площадка, сцена, автопилот и стиль
- Верхняя панель навигации и сохранения теперь остаётся на виду и убирает необходимость искать ключевые действия в длинной форме
- Настройки автопилота и стиля разнесены по отдельным карточкам, чтобы сократить визуальную перегрузку

[1.2.2]: https://github.com/Quadart21/Grok_Tg_Assistant/releases/tag/v1.2.2

## [1.2.1] — 2026-07-21

### Исправлено

- Групповой чат больше не отправляет дословные повторы и слишком похожие ответы из недавней истории
- При дубле движок делает до двух дополнительных генераций с антидубль-контекстом и отбрасывает реплику, если она всё равно повторяется
- Пример настроек дополнен параметрами окна проверки, порога похожести и числа повторных попыток генерации

[1.2.1]: https://github.com/Quadart21/Grok_Tg_Assistant/releases/tag/v1.2.1

## [1.2.0] — 2026-07-20

### Добавлено

- Вкладка **Групповой чат** умеет принимать ссылку на Telegram-чат и массово заводить в него выбранные аккаунты
- Поддержаны публичные ссылки `t.me/chat_name`, invite-ссылки `t.me/+hash` и `t.me/joinchat/hash`
- После вступления чат автоматически подставляется в селектор группового режима для дальнейшей работы

### Изменено

- Статус вступления теперь показывается по каждому аккаунту отдельно с нормализованными ошибками Telegram

[1.2.0]: https://github.com/Quadart21/Grok_Tg_Assistant/releases/tag/v1.2.0

## [1.1.0] — 2026-07-19

### Добавлено

- Вкладка **Групповой чат**: выбранные аккаунты ведут живую переписку в общем Telegram-чате по теме и ролям
- Поиск общих чатов (пересечение диалогов аккаунтов)
- Расписание активности, квоты, burst, typing, тихие паузы и веса спикеров
- Провайдер **Local** (llama.cpp / OpenAI-compatible): base URL + API key + модель
- `install_python.bat`, улучшенный `start.bat` (выбор Python 3.10–3.12 для tdata)

### Изменено

- LLM-клиент поддерживает кастомный base URL и увеличенный таймаут для локальных моделей

[1.1.0]: https://github.com/Quadart21/Grok_Tg_Assistant/releases/tag/v1.1.0

## [1.0.0] — 2026-07-02

### Добавлено

- Локальная веб-панель **Kot_Teamlead** (`127.0.0.1:8787`)
- Массовая рассылка в Telegram с AI-генерацией первых сообщений
- Автоведение диалогов (Grok, OpenAI, Gemini, Claude, DeepSeek, OpenRouter)
- Управление аккаунтами: tdata → `.session`, фильтры, массовая смена профиля
- **Пул прокси**: импорт, проверка (страна, пинг), массовые операции, привязка 1:1
- AI-агенты (ассистенты), исключённые из общей рассылки
- Роли и мастер-промпт для стиля общения
- Модульная архитектура API (`core/api/`) и фронтенда (`web/panel/js/`)

### Исправлено

- Навигация по вкладкам после рефакторинга JS
- Корректные пути `/api/*` в модульной сборке панели
